# Product Brief: AI-тьютор для Synergy LMS

Этот документ фиксирует ключевые предпосылки продукта на этапе MVP: кто пользователь, какая у него боль, какие ограничения среды и как мы будем измерять успех. Используется как единая точка правды для команды разработки, CustDev и переговоров с университетом.

## Основные параметры проекта

| Параметр | Описание |
|---------|----------|
| **Клиент** | Университет «Синергия» (Synergy LMS) |
| **Среда бизнеса** | Дистанционное высшее образование |
| **Формат продукта** | Браузерное расширение (полуинтеграция, read-only) |
| **Технический подход** | Без доступа к API LMS на этапе MVP; данные считываются с DOM-дерева интерфейса студента |

## Целевая аудитория

| Тип студента | Мотивация | Проблема | Страх | Ожидание от AI-тьютора |
|--------------|-----------|----------|-------|--------------------------|
| **Работающий студент** | Совмещать карьеру и учёбу | Нет времени, усталость, хаос в приоритетах | Не успеть / провалить дисциплины | Чёткий план, минимум решений, напоминания |
| **Студент из малого города** | Получить диплом престижного вуза | Нет поддержки, одиночество, сложности с самодисциплиной | Отстать без контроля | Структура + мягкие напоминания + ощущение сопровождения |
| **Студент с детьми** | Учиться, не нарушая уход за ребёнком | Фрагментарное время, постоянные прерывания | Потерять мотивацию или отчислиться | Гибкий план, короткие блоки обучения, чек-листы |
| **Студент с низкой самодисциплиной** | Не отчислиться, закрыть программу | Прокрастинация, хаос, не знает с чего начать | «Я всё провалю» | Жёсткая, но дружелюбная поддержка: «Сделай вот это сегодня» |
| **«Формальный» студент** | Минимум усилий ради диплома | Апатия, лень, отсутствие интереса | «Это займёт слишком много времени» | Только самое важное для сдачи: практика → тест → закрыто |

## UX-цели (измеримые)

| Цель | Как измеряем |
|------|---------------|
| Студент за 30 секунд понимает, что делать сегодня | Тестирование прототипа: % студентов, правильно ответивших на вопрос «Что твоя первая задача?» без подсказок |
| Снижение числа дней без активности >3 | Лог активности в расширении |
| Уменьшение количества просроченных дисциплин | Сравнение прогресса до/после использования |
| Рост уверенности в прохождении семестра | NPS-вопрос после недели использования: «Насколько ты уверен, что закроешь семестр в срок?» |

## Ограничения среды

| Ограничение | Последствия |
|-------------|--------------------------|
| **Нет доступа к API LMS** | Все данные — только то, что видит студент в браузере |
| **Read-only режим** | Нельзя автоматически отмечать темы как выполненные — только ручной чек или подтверждение через UI |
| **Персональные данные нельзя собирать** | Нельзя запрашивать семейное положение, доход, график работы — только то, что студент добровольно вводит (например, «сколько часов в неделю могу учиться») |
| **AI не должен работать на страницах итоговых тестов** | Расширение отключается или работает только в режиме «объяснения темы» |

## Контекст использования

- **Устройство**: ноутбук / ПК (основной), иногда мобильный браузер
- **Состояние**: усталость, отвлечённость, стресс, ограниченное время (15–30 мин)
- **Место**: дом, работа, общественный транспорт
- **Цель сессии**: быстро понять, что делать → выполнить → закрыть

## Метрики успеха MVP

| Метрика | Целевое значение |
|--------|------------------|
| % активных пользователей (≥3 сессий/неделю) | ≥60% |
| Среднее время до первого действия в сессии | ≤30 сек |
| % студентов, считающих, что «без AI было бы сложнее» | ≥40% |
| Снижение доли дисциплин с просрочкой на 2+ недели | ≥20% |
